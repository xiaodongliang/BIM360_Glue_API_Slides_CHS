回顾AcCoreConsole

  • AutoCAD 2013引入
  • 控制台 AutoCAD，无头CAD
  • DOS 命令行执行AutoCAD命令
    • 在图纸上运行一段脚本
    • 快速批处理DWG文件
  • 可加载’DBX’
  • 可加载 ‘CRX’ 插件，执行自定义命令
    • ObjectARX ，.NET ，LISP
  • 更多阅读

没有安装AutoCAD

怎么办？

AutoCAD I/O基本流程

云端AcCoreConsole 云端运行AutoCAD,批处理DWG

客户群

  • 不是最终该用户直接使用的
    • 为开发者而诞生
  • 不能交互式编辑DWG图纸
    • 交互编辑或查看DWG用AutoCAD 360 Web或Mobile
  • 不是服务器
    • 没有所谓’安装‘

接口概览

  • OData协议规范（URL 和 payloads）
  • 主要术语，3个集合
    • WorkItem
    • Activity
    • AppPackage
  • 和客户端用什么语言无关，只要能发送 HTTP请求
    • 也有一些封装好的.NET 库 ，便于使用

Activity和WorkItem

  • Activity可以理解为一个功能函数
  • WorkItem则是用来调用函数

Activity

  • 定义输入输出参数
    • 满足 URL 的参数要求

  • 计划提供Activities库
    • 作为AutoCAD专业库

  • Activities可以是公开或私有
    • 可以分享Activities

WorkItem

  • WorkItem需要
    • 关联某 ‘Activity’
    • 输入参数 (URLs)
    • 输出参数 (URLs)

  • 查看WorkItem状态:
    • 可用状态查询方式
    • 也可使用回调URL

AppPackage

  • 可上载 .NET或C++ 程序

  • CRX 程序

  • DBX 程序

  • App 必须打包为Autoloader格式

外部引用

  • DWG 文件可以有外部参照 (dwg, 图片等.)

  • 客户端必须提供所有引用
    • 网络传输package
    • 输入参数提供内联 ‘File’ 对象 (JSON 字串描述URLs和其本地名)

背后发生了什么

  • Worker通过队列收取 WorkItem

  • 下载输入参数
    • HTTP GET
  • 运行AcCoreConsole.exe

  • 将结果上载到输出参数指定的存储
    • HTTP POST/PUT
